# Use cases

## Reopen named Codex sessions

Use `codex-tabs` when you want stable names for long-lived Codex sessions instead of remembering raw session IDs.

Examples:

- reopen a personal notes session by name
- keep a work project session pinned to a short alias
- restore a known set of recurring sessions quickly

## Manage multiple Codex sessions

Use `codex-tabs` when you work across several Codex sessions and want a small registry for opening them deliberately.

Examples:

- keep separate sessions for different projects
- reopen one session or all saved sessions from the same menu
- avoid manually searching local Codex state every time

## Open saved sessions in terminal tabs

Use `codex-tabs` when you want saved Codex sessions to reopen in your terminal workflow instead of manually resuming them one by one.

Examples:

- open saved sessions in Windows Terminal tabs on Windows/WSL
- open saved sessions in `tmux` windows on Linux/macOS

## Keep discovery focused

Use `codex-tabs` when you want the add/import flow to stay clean instead of showing every old Codex session forever.

Examples:

- save the sessions you care about
- ignore other untracked previous sessions from future discovery
- keep the wizard focused on unsaved sessions only

## What codex-tabs is for

`codex-tabs` is for:

- naming persistent Codex sessions
- reopening saved sessions by alias
- organizing recurring Codex workflows in a terminal-first setup

## What codex-tabs is not for

`codex-tabs` is not:

- a multi-agent orchestrator
- a git worktree manager
- a cloud sync service
- a wrapper around remote Codex APIs
